## ‚ú® What is fancybash?

**fancybash** is a production-ready, opinionated Bash configuration designed for modern web developers. It replaces your dull terminal with a fully-featured, intelligent shell environment ‚ÄĒ without the overhead of frameworks like Oh My Zsh.

One file. One install. Zero drama.

> Built by a developer, for developers ‚ÄĒ with Node.js, Bun, Git, and Linux workflows baked in.

## ūüóĎÔłŹ Uninstall

Cleanly removes **only** the fancybash block from your config file, leaving the rest untouched:

**For Bash:**

```bash
sed -i '/# >>> fancy-bashrc >>>/,/# <<< fancy-bashrc <<</d' ~/.bashrc && source ~/.bashrc
```

**For Zsh:**

```zsh
sed -i '/# >>> fancy-zshrc >>>/,/# <<< fancy-zshrc <<</d' ~/.zshrc && source ~/.zshrc
```

**For PowerShell:**

```powershell
$p = $PROFILE; (Get-Content $p -Raw) -replace '(?s)# >>> fancy-powershell >>>.*?# <<< fancy-powershell <<<\s*', '' | Set-Content $p
```

> ūüí° Your original config remains untouched ‚ÄĒ only the `fancybash` block wrapped in `>>> / <<<` markers is removed.

## ūü¶Ą Zsh Setup Guide

Install Zsh, set it as your default shell, and supercharge it with syntax highlighting and autosuggestions ‚ÄĒ no framework needed.

### 1 ‚ÄĒ Install Git & Zsh

For Ubuntu/Debian-based systems:

```bash
sudo apt update && sudo apt install git zsh -y
```

### 2 ‚ÄĒ Verify & Set as Default

Check the installed version, then set Zsh as your default shell. Log out and back in for this to take effect.

```bash
zsh --version
chsh -s $(which zsh)
```

### 3 ‚ÄĒ Create Plugin Folder

fancybash (`config.zsh`) automatically loads plugins if they are present in the `~/.zsh/` directory.

```bash
mkdir -p ~/.zsh
```

### 4 ‚ÄĒ Install Plugins via Git

Clone both plugins directly ‚ÄĒ no plugin manager required.

```bash
git clone https://github.com/zsh-users/zsh-syntax-highlighting.git ~/.zsh/zsh-syntax-highlighting
git clone https://github.com/zsh-users/zsh-autosuggestions.git ~/.zsh/zsh-autosuggestions
```

### 5 ‚ÄĒ Restart your terminal

Open a new terminal window. fancybash will auto-source the plugins and everything will work automatically!

## ‚öôÔłŹ Font Setup (for Emoji & Icons)

fancybash uses color emoji and Fira Code ligatures in the prompt. Follow these steps for the best experience:

### 1 ‚ÄĒ Install Emoji Font

```bash
sudo apt install fonts-noto-color-emoji
```

### 2 ‚ÄĒ Install Fonts (Programming Fonts)

```bash
sudo apt update && sudo apt install -y fonts-firacode fonts-cascadia-code
```

### 3 ‚ÄĒ Set Font Priority

```bash
sudo nano /etc/fonts/local.conf
```

Paste the following config:

```xml
<?xml version="1.0"?>
<!DOCTYPE fontconfig SYSTEM "fonts.dtd">
<fontconfig>
  <alias>
    <family>monospace</family>
    <prefer>
      <family>Fira Code</family>
      <family>Noto Color Emoji</family>
    </prefer>
  </alias>

  <alias>
    <family>sans-serif</family>
    <prefer>
      <family>Noto Color Emoji</family>
    </prefer>
  </alias>
</fontconfig>
```

### 4 ‚ÄĒ Rebuild Font Cache

```bash
fc-cache -fv
```

### 5 ‚ÄĒ Set System Locale (if emoji still broken)

```bash
echo -e "LANG=en_US.UTF-8\nLC_ALL=en_US.UTF-8\nLANGUAGE=en_US" | sudo tee /etc/default/locale
sudo locale-gen en_US.UTF-8
sudo update-locale
reboot
```

## ūüďü Smart Prompt System

fancybash renders a smart two-part prompt with contextual awareness:

```
ūüöÄ myproject                          ‚Üź Emoji (auto by folder name) + colored folder
‚ĚĮ‚ĚĮ‚ĚĮ                                   ‚Üź Blinking cursor arrow (Line 2)
```

**Prompt features include:**

| Element            | Description                                                                                   |
| ------------------ | --------------------------------------------------------------------------------------------- |
| `rand_emoji`       | Folder-aware emoji ‚ÄĒ `ūüĆź` for web, `ūüüĘ` for node, `ūü•ź` for bun, `ūüźć` for py, random otherwise |
| `rand_color`       | Rainbow color cycle on every prompt render                                                    |
| `parse_git_branch` | Shows `branchname ‚Ěó` when working tree is dirty                                              |
| `cpu_temp`         | ūüüĘ Green / ūüü° Yellow / ūüĒī Red based on temperature thresholds                                 |
| `disk_usage`       | Shows free disk space on `/`                                                                  |
| `load_avg`         | System load average                                                                           |
| `get_duration`     | Shows `‚ŹĪÔłŹ Ns` for any command taking longer than 1 second                                     |
| `check_readonly`   | Shows `ūüĒí` when current directory is not writable                                             |
| `pending_updates`  | Shows `ūüÜô N` if system packages need updating                                                 |
| `battery_info`     | Shows battery % when available                                                                |
| `kernel_version`   | Displays current kernel version                                                               |

### ‚öôÔłŹ Customizing the Prompt Layout

By default, fancybash/fancyzsh renders a clean, minimalistic **single-line** prompt. However, all the dynamic system monitoring metrics listed above (such as CPU temp, disk space, active runtime versions) are already built-in and ready to be used.

You can modify or toggle the prompt layout to your liking by editing your shell configuration file (`~/.bashrc` or `~/.zshrc`) and uncommenting/commenting the lines under the **`ūüéĮ TWO LINE PROMPT`** section:

```bash
# ūüí° Uncomment these lines in your ~/.bashrc or ~/.zshrc if you want the full two-line prompt:
# PS1="\$(rand_emoji) \[\033[\$(rand_color)m\]\W\[\033[0m\] "
# PS1+="\$(folder_size) [ūüĆŅ \$(parse_git_branch)]\$(cpu_temp) \$(disk_usage) \$(load_avg) \$(get_duration) \$(check_readonly) \$(pending_updates)\n"
# PS1+="\$(node_version) ‚Ēā \$(npm_version) ‚Ēā \$(bun_version) ‚Ēā \$(kernel_version) ‚Ēā "
# PS1+="\$(time_date) ‚Ēā \$(sys_info) ‚Ēā \$(battery_info)\n"
```

Feel free to customize, add, or remove any helper functions (like `node_version`, `cpu_temp`, etc.) from your configuration block to design your own custom layout!

## ūüĖ•ÔłŹ Zed IDE Settings

Install a fully-configured `settings.json` for the [Zed](https://zed.dev) editor ‚ÄĒ works for both the **Flatpak** and **native** installations in one shot.

### One-Line Install

```bash
bash <(curl -fsSL https://gist.githubusercontent.com/rihadjahanopu/bb56371f87793c21a8fad4e4b9815a9b/raw/install-settings.sh)
```

The script will:

1. ūüíĺ **Back up** any existing `settings.json` with a timestamp
2. ‚úćÔłŹ **Write** the new config to both paths:
   - `~/.var/app/dev.zed.Zed/config/zed/settings.json` _(Flatpak)_
   - `~/.config/zed/settings.json` _(native)_
3. ‚úÖ Print a confirmation for each path

> **Restart Zed** after running the script for all settings to take effect.
